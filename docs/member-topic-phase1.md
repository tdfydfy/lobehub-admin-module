# 成员登录与自有 Topic 查看改造说明（第一阶段）

## 0. 当前状态

更新日期：2026-03-17

本阶段已完成：
- 项目成员可使用现有账号密码登录管理模块。
- 项目列表会返回成员参与的项目，不再只返回管理员项目。
- 成员进入项目后会进入只读工作台。
- 成员仅可查看自己的 Topic 统计、Topic 清单和对话详情。
- Topic 统计中，点击成员名称或 Topic 数量都可以打开清单。

本阶段仍不包含：
- CRM 客户报表
- `public.topics` / `public.messages` 原始表直查开放
- 助手详情 / 剔除
- `rt_fetch used out-of-bounds` 排障

## 1. 背景

当前管理模块最初只面向系统管理员和项目管理员。

存在的问题：
- 项目成员虽然已经在 `lobehub_admin.project_members` 中存在，但无法登录管理端。
- Topic 统计与对话详情接口全部要求项目管理员权限，成员无法查看自己的托管对话。
- 前端工作台默认按“系统管理员 / 项目管理员”两种模式组织，缺少成员只读入口。

## 2. 本轮目标

本轮仅覆盖“成员登录并查看自己的托管 Topic / 对话详情”。

目标包括：
- 允许项目成员登录管理模块。
- 允许项目成员看到自己所属项目。
- 允许项目成员在项目内查看自己的 Topic 统计。
- 允许项目成员查看自己的 Topic 列表和对话消息详情。
- Topic 统计页中，“成员名称”和“Topic 数量”都可以打开 Topic 清单。

## 3. 本轮不做

以下内容明确不纳入本轮：
- `crm.customer_profiles` 维度的“销售看自己的客户报表”
- 数据查看页对 `public.topics` / `public.messages` 的原始表直查开放
- 助手详情查看、助手剔除接口
- 批量托管任务对管理员角色的扩展覆盖
- `rt_fetch used out-of-bounds` 的数据库扩展问题修复

说明：
- 成员查看 Topic 的入口统一走“对话统计 -> Topic 清单 -> 消息详情”。
- 不通过“数据查看”页直接开放底层表。

## 4. 权限策略

本轮按现有角色模型推进，不新增 `sales` 第三角色。

沿用角色：
- `admin`：项目管理员
- `member`：项目成员 / 销售

权限边界：
- 系统管理员：保持现有全部能力。
- 项目管理员：保持现有项目管理能力。
- 项目成员：仅允许只读访问自己在所属项目内的 Topic 相关数据。

成员不具备：
- 成员管理
- 模板设置
- 助手批量配置 / 刷新
- 数据查看页访问
- 成员运营报表访问

## 5. 后端改造范围

### 5.1 认证与上下文

登录准入逻辑已调整为：
- 系统管理员可登录
- 至少属于 1 个项目的用户可登录

上下文中已区分：
- `managedProjectCount`
- `joinedProjectCount`

### 5.2 项目访问

项目列表已改为返回当前用户参与的全部项目，而不是仅返回其担任 `admin` 的项目。

项目级访问分两层：
- `project admin`：可做管理操作
- `project member`：只读访问自己的 Topic 数据

### 5.3 Topic 接口

以下接口已支持成员访问，但成员只能访问自己的数据：
- `GET /api/projects/:projectId/reports/topic-stats`
- `GET /api/projects/:projectId/reports/topic-stats/users/:userId/topics`
- `GET /api/projects/:projectId/reports/topic-stats/topics/:topicId`

约束规则：
- 项目管理员 / 系统管理员：可看项目范围内数据。
- 项目成员：只能看 `actor.id` 对应的数据。

## 6. 前端改造范围

已新增“成员只读工作台”模式：
- 非系统管理员登录后，会根据当前所选项目内的角色决定是“项目管理员工作台”还是“成员只读工作台”。

成员只读工作台只保留：
- 项目基础信息
- 对话统计

成员只读工作台不展示：
- 成员管理
- 助手配置
- 数据查看
- 数据报表

## 7. 交互改造

Topic 统计页当前支持：
- 点击成员名称：打开 Topic 清单
- 点击 Topic 数量：打开 Topic 清单

保留现有弹窗链路：
- 成员列表
- Topic 清单
- 对话消息详情

## 8. 验收口径

验收通过标准：
- 项目成员可使用现有账号密码登录管理模块。
- 登录后能进入自己参与的项目。
- 成员进入项目后可以看到“对话统计”页。
- 成员只能看到自己的 Topic 统计、Topic 列表和消息详情。
- 成员无法打开成员管理、助手配置、数据查看、数据报表。
- 项目管理员原有管理能力不回退。
